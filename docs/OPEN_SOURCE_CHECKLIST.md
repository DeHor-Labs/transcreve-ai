# Open Source Checklist

Before making the repository public:

- [ ] Rotate any API key that has appeared in chat, terminal history, screenshots or logs.
- [ ] Confirm `.env` is ignored and not present in Git history.
- [ ] Confirm `outputs/`, videos, audio, frames and real user data are ignored.
- [ ] Keep examples synthetic or explicitly permissioned.
- [ ] Run tests locally.
- [ ] Confirm the GitHub Actions CI passes.
- [ ] Confirm `LICENSE`, `SECURITY.md`, `CONTRIBUTING.md` and README are up to date.
- [ ] Decide if the repository should include a demo video or only generated synthetic fixtures.
- [ ] Set GitHub topics and description.
- [ ] Only then switch visibility from private to public.
